* **Problem**: 이번 프로젝트에서 아쉬웠던 것, 발생한 버그, 커뮤니케이션 미스, 시간 부족 등 해결했어야 할 어려움.

 ( 김태형  )  
* **Keep**: 실시간 인물 인식 결과를 로봇의 이동 제어와 연계하여 카메라 중심에 사용자가 위치하도록 구현하였다.
단순 이동 로봇이 아닌 환경 변화에 반응하는 폐루프(Closed-loop) 제어 시스템을 구축하였다.
* **Problem**: 사용자가 너무 빠르게 움직이거나 여러 사람이 동시에 이동하는 경우 목표 인물을 안정적으로 추적하지 못하는 경우가 있었다.
* **Try**: 장애물 회피 기능과 경로 계획 기능을 추가하여 실제 서비스 로봇 수준의 자율성을 확보한다.

 (구민서)  
* **Keep**:
스마트폰 웹앱과 ROS 노드의 연동을 지속적으로 테스트하며 통신 오류를 빠르게 확인하였다.
다인 인식, 자동 촬영, TTS, 카운트다운, 갤러리 업로드 등 기능을 단계적으로 구현하여 안정성을 높였다.
문제 발생 시 ROS Topic, Node, Log를 확인하면서 원인을 분석하고 해결하는 과정을 반복하여 통합 능력을 향상시켰다.
* **Problem**:
MediaPipe와 FaceLandmarker의 특성으로 인해 전신 촬영에서는 얼굴 인식률이 낮아 미소 판정의 정확도가 떨어졌다.
다인 촬영 시 사람 선택 기준과 그룹 중심 계산이 명확하지 않아 로봇이 여러 사람 사이를 반복해서 추적하는 문제가 발생하였다.
TTS가 반복 재생되거나 카운트다운과 촬영 타이밍이 맞지 않아 사용자 경험이 떨어지는 문제가 있었다.
프로젝트 후반부에 통합 과정에서 발생한 버그 수정에 많은 시간을 사용하여 충분한 성능 튜닝을 진행하지 못하였다.
* **Try**:
사람 추적 알고리즘에 ID Tracking(DeepSORT, ByteTrack 등)을 적용하여 동일한 사람을 안정적으로 추적하도록 개선해 보고 싶다.
미소 판정을 FaceLandmarker뿐만 아니라 얼굴 검출 결과와 함께 사용하여 다양한 거리에서도 정확도를 높이고 싶다.
ROS State Machine을 보다 체계적으로 설계하여 TTS, 카운트다운, 촬영이 순차적으로 동작하도록 개선하고 싶다.
프로젝트 초기부터 단위 테스트와 통합 테스트를 병행하여 후반부의 디버깅 시간을 줄이고, 실제 환경에서 반복 테스트를 통해 시스템의 안정성을 높이고 싶다.
